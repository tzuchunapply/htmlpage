Defect Identification and Classification Using YOLOv5 and Azure Machine Learning

Hackathon Project | University of Washington

Project Objective

The goal was to accurately detect and localize defects in MoS₂ (Molybdenum disulfide) images and classify them into multiple categories using deep learning.

Azure ML YOLOv5 Roboflow ImageJ Python
  • Object Detection: Localize defects by drawing bounding boxes.
  • Classification: Identify defect types (Broken Region, Vacancy, Replacement).
  • Preprocessing: Enhance raw microscopy images for higher accuracy.

Methodology & Pipeline

1. Data Preparation

Used ImageJ for initial labeling and Roboflow to generate annotation files (.txt) readable by YOLOv5.

# YAML Configuration Snippet
nc: 3
names: ['Broken Region', 'Replacement', 'Vacancy']

2. Model Training

Trained a multiclass classification model using Azure Machine Learning resources, leveraging YOLOv5's architecture for high-speed detection.

Performance Analysis

Model Limitations

  • F1 Score: 0.344 (Initial iteration)
  • Human Error: Subjective labeling in noisy images.
  • Small Dataset: Need for more diverse samples.
  • Label Purity: Vacancy and Replacement labels tend to overlap.

Preprocessing & Verification

Image Preprocessing

Applied various filters to enhance defect visibility:

  • Filters: Gaussian, Median, and Uniform filters.
  • Color Mapping: "Ice" map for better contrast (~8.71 contrast ratio).

Verification via Intensity Plot

To verify model predictions, Intensity Line Profiles were generated. The peaks and valleys in the intensity plot accurately confirmed the presence of broken regions versus vacancies.

[Intensity Plot Visualization]

Conclusions & Future Improvements

The model successfully detected and classified different types of defects even in images sourced from literature, proving its generalizability.

  • Incorporate more background images to reduce false positives.
  • Use data augmentation to expand the small dataset.
  • Apply secondary verification using intensity profiles for edge cases.
